AMF project configuration

This tutorial covers installing all the tools you'll need to run AMF in your application.

Artifacts to use AMF can be obtained from:

  • Maven
  • NPM

Add AMF to your dependencies#

First you need to add AMF to your project, whether in Gradle, Maven, or NPM.

  • Gradle
  • Maven
  • NPM
// add mulesoft repository
repositories {
maven {
url 'https://repository-master.mulesoft.org/nexus/content/repositories/releases'
}
}
dependencies {
compile 'com.github.amlorg:amf-client_2.12:x.y.z'
}
<dependency>
<groupId>com.github.amlorg</groupId>
<artifactId>amf-client_2.12</artifactId>
<version>x.y.z</version>
</dependency>

NPM:

$ npm install --save amf-client-js

Yarn:

$ yarn add --save amf-client-js

NOTE: you may use the -SNAPSHOT versions of the artifacts at your own risk since those snapshot versions may contain breaking changes.

Code setup#

To use AMF you must first initialize it like so:

  • Java
  • JavaScript
import amf.client.AMF;
class App {
public static void main(String[] args){
AMF.init().thenApply(() -> {
// ... your code
});
}
}
const amf = require('./lib/amf-client-module.js')
await AMF.init();
// ... your code

That's it! this is everything you need to start using AMF, in the following sections you can learn how to use every feature AMF has.
